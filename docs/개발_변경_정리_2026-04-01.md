# 개발 변경 정리 (2026-04-01)

- 브랜치: `UI_stitch`
- 대상: `src/CameraDemo` 중심 UI/동작 안정화 및 저장 경로 개선

## 2. 확정된 운영 정책
- 파이프라인: `NV12 -> NV12 (Fixed)`
- 목표 FPS: `24fps`
- 기본 해상도: `3840x2160`
- 녹화 시작 가드: Negotiated 서브타입 `NV12`, FPS `>= 23.5`

## 3. UI 확정 사항
- 타이틀: `Q-Camera : Sony 13MP - IMX258`
- Preview 정지 시 중앙 오버레이: `PREVIEW STOPPED`
- 녹화 중 중앙 오버레이: `REC`
- 저장 중 중앙 오버레이: `SAVING...` + `done/total` 진행률
- 저장 중 전체 메뉴 잠금(상단/우측/저장 패널)
- Camera Controls의 `Negotiated` 텍스트를 `Resolution` 표시로 변경

## 4. 저장 멈춤 대응
- 폴더 선택 대화상자(`FolderBrowserDialog`) 제거
- 자동 저장 폴더 생성 방식으로 변경:
  - `saved_frames/save_yyyyMMdd_HHmmss_fff`
- 저장 인코딩은 STA 전용 스레드에서 수행
- Preview 정지 timeout 시 저장 강행 금지(중단 후 사용자 안내)
- 로그 업데이트 부하 완화:
  - `AppendText` 사용
  - 최대 로그 길이 제한 적용
  - 진행률 로그 간격 완화
