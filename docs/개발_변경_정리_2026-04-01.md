# 개발 변경 정리 (2026-04-01)

## 1. 범위
- 브랜치: `codex/feature-validation`
- 대상: `src/CameraDemo`, `src/CameraCli`, `src/CameraSDK`, `src/CameraCore` 기준 1단계 안정화 및 검증

## 2. 확정된 운영 정책
- 파이프라인: `NV12 -> NV12 (Fixed)`
- 목표 FPS: `24fps`
- 기본 해상도: `3840x2160`
- 녹화 시작 가드: Negotiated 서브타입 `NV12`, FPS `>= 23.5`

## 3. UI 확정 사항
- 타이틀: `Q-Camera : Sony 13MP - IMX258`
- Preview 정지 시 중앙 오버레이: `PREVIEW STOPPED`
- 녹화 중 중앙 오버레이: `REC`
- 저장 중 중앙 오버레이: `SAVING...` + `done/total` 진행률
- Recording/Save 차단 사유를 중앙 경고 오버레이로 표시
- 저장 중 전체 메뉴 잠금(상단/우측/저장 패널)
- Camera Controls의 `Negotiated` 텍스트를 `Resolution` 표시로 변경

## 4. 저장/녹화 제어 개선
- 저장 위치 선택 대화상자(`FolderBrowserDialog`) 사용
- 저장 가능 조건: 녹화 후 미저장 캡처 데이터가 있을 때만
- 저장 데이터가 없으면 저장 차단 + 중앙 경고 + 로그 기록
- Auto Exposure ON 상태에서는 Recording 시작 차단
- Auto Exposure OFF 시 마지막 수동 Exposure 값으로 즉시 복귀
- Auto Exposure ON 상태에서는 Gain 수동 변경 비활성화, OFF 시 수동 Gain 재적용
- 저장 인코딩은 STA 전용 스레드에서 수행
- Preview 정지 timeout 시 저장 강행 금지(중단 후 사용자 안내)

## 5. 해상도 변경 안전장치
- Preview가 동작 중이거나 저장/녹화 중이면 해상도 변경 차단
- 저장 중 내부 pause 상태에서도 해상도 변경 차단 유지
- 사용자 `Stop Preview` 성공 상태에서만 해상도 변경 허용
- 조건 미충족 시 콤보 선택 즉시 이전 값으로 롤백 + 로그 기록

## 6. CLI/SDK 검증 보강
- `CameraCore.dll`에 `Camera_GetProcAmpValue` export 추가
- `CameraSDK`에 ProcAmp readback 래퍼 추가
- `CameraCli`에 `--probe-sharpness` 추가
- Sharpness 패턴 검증 결과: `0 -> 128 -> 255` 순으로 edge metric 증가 확인

## 7. 확인 포인트
- 저장 결과 폴더 선택/완료 여부
- 저장 중 오버레이 및 진행률 표시
- 저장 완료 후 Preview 자동 복귀 여부
- Auto Exposure ON 상태에서 Recording 차단 여부
- AEC ON/OFF 전환 시 수동 Exposure/Gain 복귀 여부
- Sharpness CLI 검증(`--dump-procamp`, `--probe-sharpness`) 결과
- 사용자 Stop Preview 상태가 아니면 Resolution 변경 차단 여부
- `camera_demo.log`의 차단 로그/저장 로그 기록 여부
