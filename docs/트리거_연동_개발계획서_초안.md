# ETIO-2444 트리거 신호 연동 개발 계획서 (초안)

작성일: 2026-04-01  
브랜치: `codex/trigger-signal-integration`

## 0. 전제 및 현재 확인 결과

### 0.1 문서 검토 상태
- 로컬 PDF 2종(`B2S_ETIO-2444_Product brief.pdf`, `B2S_ETIO-2444_User Guide.pdf`)은 현재 Codex 실행 환경에서 본문 텍스트 추출이 제한되어, 문서 원문 세부 파라미터(레지스터/명령 포맷)까지는 기계적으로 확보하지 못함.
- 다만, 공개 소개 페이지 기준으로 ETIO-2444는 아래 특성을 가짐.
  - USB 기반 호스트 제어
  - Encoder 입력 기반 Trigger 생성/분배
  - Isolated Digital I/O 제공
  - Trigger 출력(Line Driver) 중심 장치

### 0.2 현재 Q-Camera SDK 코드 상태
- `src/CameraCore/CameraCore.cpp`
  - 내부 변수 `g_trigger_mode` 존재 (연속/소프트웨어/하드웨어 개념 흔적)
  - 그러나 외부 SDK export API(`CameraSDK.h`, `CameraSDK.cs`)에는 Trigger 모드 설정/실행 API가 없음.
- `src/include/CameraSDK.h`, `src/CameraSDK/CameraSDK.cs`
  - Start/Stop, Exposure/Gain/Focus, ProcAmp, 모드/해상도 설정 등만 제공
  - ETIO-2444 같은 외부 트리거 보드 제어 API 부재
- 결론: **현재 버전만으로 ETIO-2444 트리거 신호 연동 제어는 불가**

## 1. 질문: 실물 ETIO-2444 없이 기능 검증 가능한가?

## 1.1 가능한 범위 (실물 없이)
- 가능: 소프트웨어 아키텍처/인터페이스 개발
  - Trigger Board 추상화 계층 설계
  - Mock Driver(가상 보드) 구현
  - 카메라 녹화/저장 파이프라인과 Trigger 이벤트 연결 로직 개발
  - UI/로그/오류 처리 시나리오 검증
- 가능: 단위 테스트/통합 테스트(가상 이벤트 기반)
  - Rising/Falling edge 이벤트 주입
  - 주기 Trigger, burst Trigger, timeout, 장치 연결 해제 시나리오

## 1.2 불가능한 범위 (실물 없이)
- 불가: 물리 신호 레벨 검증
  - Line Driver/절연 I/O 전기적 레벨
  - 실제 트리거 지터/지연 시간(ns~us)
  - 장치 드라이버/펌웨어 고유 동작 검증
- 불가: USB 실제 통신 프로토콜 완전 검증
  - 벤더 명령 프레임/응답 코드의 실제 동작 확인

## 1.3 결론
- **실물 없이도 소프트웨어 개발(구조/로직) 70~80%는 진행 가능**
- **최종 기능 인증(하드웨어 트리거 품질 검증)은 실물 ETIO-2444 필수**

## 2. 목표 정의

1. ETIO-2444 트리거 입력/출력 상태를 Q-Camera에서 읽고 제어
2. Trigger 기반으로 카메라 Capture/Record 시작/정지 제어
3. Trigger 이벤트와 프레임 타임스탬프를 로그로 남겨 재현 가능성 확보
4. 실물 부재 기간에는 Mock 기반으로 선개발, 실물 입수 후 빠른 검증 전환

## 3. 개발 아키텍처 제안

## 3.1 계층 분리
- `ITriggerController` (공통 인터페이스)
  - Connect/Disconnect
  - ConfigureTrigger(...)
  - Arm/Disarm
  - ReadInputState / SetOutputState
  - OnTriggerEdge 이벤트
- `Etio2444Controller` (실물 드라이버)
  - USB 통신 벤더 명령 구현
- `MockTriggerController` (가상 드라이버)
  - 타이머 기반 edge 이벤트 생성

## 3.2 카메라 연동 방식
- `TriggerOrchestrator` 추가
  - Trigger edge 수신
  - Recording 상태 머신 제어
  - 중복 트리거 debounce / cool-down / timeout 처리

## 3.3 UI 연동
- 우측 패널에 Trigger 섹션 추가(브랜치 내 기능)
  - Controller: Mock / ETIO-2444
  - Trigger Source: Encoder / DI
  - Edge: Rising / Falling
  - Arm/Disarm
  - Trigger Count / Last Trigger Time

## 5. 테스트 계획

## 5.1 실물 없음(Mock)
- TC-M1: 10Hz rising edge -> record start 1회
- TC-M2: burst 100개 -> debounce 정상 동작
- TC-M3: disconnect event -> 안전 중지 및 경고

## 5.2 실물 있음(ETIO-2444)
- TC-H1: DI 입력 edge 인식 정확도
- TC-H2: Encoder trigger mode 동작
- TC-H3: Trigger-Frame 타임 차 측정
- TC-H4: 장시간(30분+) 연속 안정성
